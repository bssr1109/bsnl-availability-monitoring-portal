# BSNL Availability Monitoring & Improvement Portal

Mobile-friendly Next.js + TypeScript demo app for daily BTS outage monitoring, Field Officer remarks, EOD submission, improvement proposals, 98% availability tracking, analytics, and Excel import/export.

## Run locally

```bash
npm install
npm run sample:excel
npm run dev
```

Open `http://localhost:3000`.

## Demo mode

The app starts without Supabase. Data is seeded from `lib/seed.ts` and saved to browser `localStorage` through `LocalJsonRepository` in `lib/repository.ts`. Use the header buttons to simulate:

- AGM/Admin
- Field Officer ETURNAGARAM
- Field Officer MULUGU
- Field Officer NARSAMPET
- Field Officer PARKAL
- Management Viewer

## Included workflows

- AGM/Admin Excel upload with flexible column matching, including compact BSNL export headers such as `bts_ip_id`, `bts_name`, `bts_down_dt`, `bts_up_dt`, `dur`, `alarm_code`, and `addl_info`.
- BTS identity is based on `bts_ip_id`; uploaded BTS IP IDs that are not in demo seed data are automatically added as sites and mapped to the Field Officer for their SDCA.
- BTS master data upload uses `Site ID` as the same key as `bts_ip_id` and stores the responsible Field Officer/person from Slot 1.
- Raw alarm row preservation and consolidated incident creation.
- Duplicate upload protection by upload fingerprint.
- Field Officer-only site visibility by SDCA mapping.
- Remarks and improvement proposal entry are restricted to Field Officer logins.
- AGM/Admin can reopen a submitted Field Officer day with a reason so the Field Officer can correct remarks and resubmit EOD.
- AGM/Admin has a dedicated Improvement Proposals view with an Excel download.
- AGM/Admin has a dedicated BTS Master Data view with upload, matching status, and Excel download.
- Outage remarks form with major-outage and delay-reason validation.
- Improvement proposal assessment with mandatory rule detection.
- EOD validation, submission, late flag, and record locking.
- Availability math with overlap-safe downtime merging.
- Recharts analytics for SDCA, root cause, top downtime sites, and daily trend.
- Excel downloads for detailed remarks, site report, pending remarks, proposals, EOD, temporary restorations, below-target sites, and single-site workbooks.

## Files to notice

- `lib/types.ts` contains the application data interfaces.
- `lib/repository.ts` keeps local JSON and future Supabase repositories modular.
- `lib/availability.ts` contains downtime interval merging and 98% availability math.
- `lib/excel.ts` contains SheetJS import/export and incident grouping logic.
- `public/sample-outage-upload.xlsx` is generated by `npm run sample:excel`.
